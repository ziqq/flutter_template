# Interaction with localization, generation, rules

## Source of truth (authoritative)
All localization keys and values come from a dedicated **Google Sheet**
(Excel-like table). Any change MUST be done in the sheet, not in the repo.

This repository treats `packages/localization/**` as a generated artifact:
no manual edits under this folder are allowed.

Important: this includes any ARB files under `packages/localization/**`.
They are generated outputs too.

## Where the sheet is configured (authoritative)
The Google Sheet ID and all generation parameters are configured in the VS Code
task definition: `.vscode/tasks.json` → task: `dart:sheety_localization:generate`

The sheet ID is passed via: `--sheet=<SHEET_ID>`

The task is also authoritative for:
- working directory (`cwd`),
- credentials file (`--credentials=...`),
- output folders (`--lib`, `--arb`, `--gen`),
- ignore rules (`--ignore=...`),
- formatting flags (`--format`, `--no-last-modified`),
- naming (`--prefix=...`).

Do not duplicate the full command line in docs/README. If parameters change,
update the task definition and treat it as the single source of truth.

## Generated outputs (do not edit)
The following outputs are generated by the localization toolchain and MUST NOT
be edited manually:
- `packages/localization/**` (ALL files, without exceptions)
- `**/*.g.dart`
- `**/*.gen.dart`
- `**/generated/**`
- `**/*.mocks.dart`

If you need to change a string/key:
1) update the Google Sheet,
2) run generation,
3) commit the regenerated outputs.

## Using localization in the app
Preferred access patterns:
- Use `BuildContext` extensions from: `lib/src/common/utils/context_extension.dart`

Deprecated layer:
- `lib/src/common/localization/**` exists but should not be extended unless explicitly required (see README notes).

## Generation (the only allowed way to update package/localization)
Run the VS Code task: `.vscode/tasks.json` → `dart:sheety_localization:generate`

Recommended workflow:
1) Update the Google Sheet
2) Run the task (`Cmd+Shift+P` → Tasks: Run Task → `dart:sheety_localization:generate`)
3) Review git diff
4) Commit generated changes

## Credentials
The generation task uses a credentials file (`--credentials=credentials.json`)
from the `packages/localization` working directory.

Do not embed secrets into generated code. Keep credentials handling aligned
with repo/CI policy.

## Troubleshooting checklist
- Ensure dependencies are fetched (`fvm flutter pub get`)
- Ensure the VS Code task runs from repo root (task `cwd` is set by config)
- If generation fails: check the task definition in `.vscode/tasks.json`
